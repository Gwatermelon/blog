论文研读：Fast Inference from Transformers via Speculative Decoding

投机解码经典工作：用小模型生成草稿，由大模型并行验证，同时保持输出分布不变。

一句话结论#

用更便宜的草稿模型一次提出多个候选 token，再让目标模型通过一次并行前向完成验证；借助拒绝采样修正，可以在获得加速的同时保持目标模型的输出分布。

要解决的问题#

自回归生成具有严格的数据依赖，每一步通常只产生一个 token。即使单次前向没有吃满硬件，下一步也必须等待上一 token 生成完成，导致内存访问和调度开销难以摊薄。

核心机制#

  1. 草稿模型连续生成若干候选 token；
  2. 目标模型并行计算这些位置的概率分布；
  3. 按顺序接受满足条件的候选；
  4. 第一个拒绝位置从修正后的分布重新采样；
  5. 若候选全部通过，再从目标模型多采样一个 token。

研读问题#

  • 接受率与两个模型分布距离之间有什么定量关系？
  • 草稿长度如何随 batch size、模型规模和硬件变化？
  • 为什么“减少目标模型调用次数”不必然带来同等比例的端到端加速？
  • 在确定性解码与随机采样下，验证过程有哪些差异？

复现计划#

  • 实现单 batch 的基础投机采样；
  • 验证输出分布与目标模型一致；
  • 记录接受率、平均接受长度、TPOT 与显存占用；
  • 扫描不同草稿长度和草稿/目标模型组合；
  • 分析性能瓶颈位于计算、访存还是框架调度。
